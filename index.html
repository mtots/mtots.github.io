Mtots is...

A programming language with 3 core goals.

None of these goals are new - in fact, each goal is isnpired by a mainstream programming language that achieves the goal incredibly well.

The novel aspect of Mtots is that it combines these goals into a single language.

  • Portable

    Like Lua

    In my opinion, a truly portable language should be easy to bend, easy to modify, and easy to port. Of the mainstream programming languages out there, I believe Lua is the only one that really achieves this sort of portability, and it does so with two design choices that mtots adopts.

    • The implementation of the core language is a set of C89 compliant C source and header files that you can easily embed in any C or C++ project by copying over these files.
    • The design of the language's syntax and semantics are carefully crafted to keep the implementation simple. Example (1) Lua's grammar is designed to work well with a single pass compiler. Example (2) Lua's object model is incredibly simple - aside from a fixed collection of builtin primitive types, all objcts are tables.

  • Fun

    Like Python

    In my opinion, a fun language is one that is easy for beginners to learn, and delight for experts to play with. The mainstream language that I believe does this best is Python.

    • Syntax, function, class and module names in the standard library are heavily borrowed from Python

  • Statically Typed

    Like Typescript

    In my opinion, Typescript's static typing is really delightful. Mtots steals some ideas from Typescript.

    • Language grammar includes first class support for type annotations even though the interpreter may ignore all type annotations
    • The maintainers of Typescript also maintains a vscode extension that exemplifies what a great development experience with the language is like. The vscode extension for Mtots is also maintained in the Mtots repository here.

Mtots also has the following secondary goals.

  • (Potentially) Fast

    Like Java

    If the language is largely statically typed, it opens up the possibility of a language implementation with a performant optimizing compiler. I think a well designed type system should allow for this. However, a runtime that takes advantage of static typing would be significantly more complex and it just doesn't make sense for me to do right now. But I want to keep that door open - so the design of the language's type system will keep this in mind.

  • Traits

    Like Rust

    Use traits over inheritance. Of course in Mtots, traits will get no runtime support, so they cannot work exactly the same way as in Rust. But I plan on Mtots adopting some sort of traits into its type system rather than inheritance (this is still a TODO).

    This is not really a "goal", but it is an important feature I do not want to forget about, so I am temporarily listing it as a goal here.
